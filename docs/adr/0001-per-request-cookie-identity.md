# Per-request cookie identity, replacing the single-seat current-user file

`users.py` originally kept exactly one "current user" for the whole running server process, persisted in `.yani/current_user.txt` and read by every request. That was deliberate for local/personal use, but it means two browsers (or two students on one household's devices) hitting the server at once share one identity — logging in as someone else in one tab silently switches every other tab's identity mid-conversation. We're moving to genuine concurrent multi-user: each browser now gets its own identity via an `httponly`, long-lived cookie (`yani_user`), read into a per-request contextvar by middleware at the top of every request (`server.py`), so concurrent requests from different learners never see each other's state.

The cookie's value is the learner's slug directly — there is no server-side session table mapping an opaque token to a slug. We considered that more conventional approach and rejected it: there's no authentication in this app (typing a name is enough to resume that learner's home, same as before), so an opaque token buys no real security here, and it would need to survive the dev server's `reload=True`, which wipes in-memory state on every save — meaning the token table would need its own file-backed persistence for no corresponding benefit. We also considered carrying identity via a client-managed `localStorage` token attached as a manual header on every `fetch()`, but a cookie needs no changes to the existing `/chat`, `/action`, `/history` call sites (the browser attaches it automatically), where a header would need every call site touched.

We explicitly did not add real authentication (password/PIN) as part of this change — the goal is isolation between concurrent browsers, not access control, and nothing in this app's scope (a small, known set of learners) called for it. A "log out" button was added specifically to handle a shared device, since the cookie is long-lived and there was previously no way to switch identity within one browser short of clearing cookies manually.
